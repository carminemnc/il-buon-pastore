# i18n — Bilingual System

## Overview

Client-side internationalization with two languages: **Italian** (default) and **English**. No external library — custom implementation in `docs/js/i18n.js`.

---

## Dictionary Structure

Flat object with dot-notation keys, one per language:

```js
const DICT = {
  it: {
    'hero.title': 'Il Buon Pastore',
    'pascolo.p1': 'Tutto inizia dall\'erba...',
    // ...
  },
  en: {
    'hero.title': 'Il Buon Pastore',
    'pascolo.p1': 'It all starts with the grass...',
    // ...
  }
};
```

---

## Key Naming Convention

| Pattern | Usage | Example |
|---------|-------|---------|
| `{section}.title` | Section heading | `pascolo.title` |
| `{section}.label` | Uppercase label above heading | `pascolo.label` |
| `{section}.p1` | First paragraph | `metamorfosi.p1` |
| `{section}.p2` | Second paragraph (optional) | `stagionatura.p2` |
| `nav.{id}` | Navigation link text | `nav.galleria` |
| `hero.{field}` | Hero-specific fields | `hero.cta`, `hero.scroll` |
| `contatti.{field}` | Contact section fields | `contatti.dove`, `contatti.orari.label` |
| `galleria.{field}` | Gallery section | `galleria.hint` |
| `meta.desc` | Meta description tag | — |

---

## DOM Binding

### Text content
```html
<h2 data-i18n="pascolo.title"></h2>
```
Updated via `el.innerHTML = dict[key]` — supports HTML tags in values (e.g. `<br>`, `<a>`).

### Attribute
```html
<meta name="description" data-i18n-attr="content" data-i18n="meta.desc" content="...">
```
When `data-i18n-attr` is present, sets that attribute instead of innerHTML.

---

## Bilingual Fields in Manifests (main.js)

Product data and contact info use object notation:

```js
{ it: 'Cremosa', en: 'Creamy' }
```

Resolved at runtime by the `t(field)` helper:
```js
function t(field) {
  return typeof field === 'object' ? (field[lang()] || field.it || '') : (field || '');
}
```

## API (window.i18n)

| Method | Description |
|--------|-------------|
| `i18n.setLang('en')` | Apply language, update all `[data-i18n]` elements, persist |
| `i18n.toggle()` | Switch between IT and EN |
| `i18n.current()` | Returns current language (`'it'` or `'en'`) |
| `i18n.onLangChange(cb)` | Register callback, receives `newLang` string |
| `i18n._dict` | Reference to IT dictionary (used for key existence checks) |

---

## Persistence

- **Key**: `ibp-lang` in `localStorage`
- **Detection order**: localStorage → `navigator.language` (first 2 chars) → fallback `'it'`
- Applied on page load before any DOM generation in `main.js`

---

## Language Change Flow

1. User clicks lang toggle → `i18n.toggle()` called
2. `apply(newLang)` iterates all `[data-i18n]` elements, sets innerHTML/attribute
3. Sets `document.documentElement.lang` and `document.body[data-lang]`
4. Persists to localStorage
5. Fires all registered `onLangChange` callbacks
6. In `main.js` callback: orari section is rebuilt with `buildOrari()` (because it uses `t()` on object fields), then label text set manually

## How to Add/Modify Text

1. Add key in both `DICT.it` and `DICT.en` in `i18n.js`
2. Reference it with `data-i18n="your.key"` in HTML or generated DOM
3. If the text contains HTML (links, `<br>`), it works — innerHTML is used
4. For product fields, use `{it:'...', en:'...'}` in the GALLERY/CONTATTI objects in `main.js`

---

## Important Notes

- Empty string values are valid (e.g. `tavola.p2: ''`) — `p:empty{display:none}` hides them
- The `p2` tag is only generated if the key exists in `_dict` (checked at DOM generation time)
- Language toggle CSS: `body[data-lang="it"] .lang-it { opacity:1 }` highlights active language
- All product names are invariant (Italian proper names) — not translated
